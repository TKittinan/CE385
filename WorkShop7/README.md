1. ส่วนของ User (จัดการผู้ใช้)
หา User จาก Email (GET /users/email/:email): ตัวนี้ผมใช้ findUnique เพราะ Email มันซ้ำไม่ได้อยู่แล้ว พอกรอก email ใน URL ปุ๊บ ระบบจะวิ่งไปเช็คในฐานข้อมูลว่ามีใครใช้เมล์นี้ไหม ถ้ามีก็ดึงข้อมูลคนนั้นออกมาโชว์ครับ

ลบ User (DELETE /users/:id): อันนี้ใช้ delete โดยอ้างอิงจาก userid ที่เป็น UUID ครับ แค่ก๊อปไอดีจากหน้าเว็บมาวางใน URL แล้วสั่งลบ ข้อมูลคนนั้นก็จะหายไปจากเบสทันที

2. ส่วนของ Post (จัดการโพสต์/บทความ)
สร้างโพสต์ใหม่ (POST /posts): เวลาจะลงบทความใหม่ ต้องส่งชื่อเรื่อง (title) เนื้อหา (content) และที่สำคัญคือต้องมี authorId ด้วย เพื่อให้รู้ว่า User คนไหนเป็นคนโพสต์ครับ (เป็นการเชื่อมความสัมพันธ์ 1:Many)

ดูโพสต์ทั้งหมด (GET /posts): ดึงทุกอย่างที่อยู่ในตาราง Post มาโชว์ และผมสั่ง include author ไว้ด้วย เพื่อให้มันดึงชื่อคนเขียนติดมาโชว์พร้อมกันเลย ไม่ต้องไปกดหาแยก

ดูโพสต์รายอัน (GET /posts/:id): ใช้ค้นหาบทความเดียวๆ โดยใช้ postid ครับ เหมาะสำหรับเวลาเราคลิกเข้าไปอ่านเนื้อหาเต็มๆ ของบทความนั้น

แก้ไขโพสต์ (PUT /posts/:id): ถ้าเขียนผิดก็สั่งแก้ได้ โดยระบุไอดีโพสต์ แล้วส่งข้อมูลใหม่ไปทับ อันนี้จะใช้คำสั่ง update ของ Prisma ครับ

ลบโพสต์ (DELETE /posts/:id): ทำงานเหมือนลบ User เลยครับ คือระบุไอดีบทความที่ต้องการทิ้ง แล้วสั่งลบออก
