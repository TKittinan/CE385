ใน Workshop 5.1 ได้สร้าง endpoint สำหรับแสดงข้อมูลนักศึกษาทั้งหมด แสดงข้อมูลตาม ID เพิ่มข้อมูลใหม่ แก้ไขข้อมูล และลบข้อมูล โดยมีการส่ง HTTP Status Code ให้เหมาะสม เช่น 200, 201 และ 404

ใน Workshop 5.2 ได้เพิ่ม Middleware เพื่อใช้ตรวจสอบข้อมูลก่อนทำการเพิ่มหรือแก้ไขนักศึกษา โดยตรวจสอบว่าใน request body มี name และ age หรือไม่ หากข้อมูลไม่ครบจะส่งสถานะ 400 พร้อมข้อความ “Invalid data” แต่ถ้าข้อมูลถูกต้องจะใช้ next() เพื่อส่งต่อไปยังขั้นตอนถัดไป
